# TrivaaLogic Ltd.

Company website.

## Landing page structure

* Title: "TrivaaLogic - custom software for web, mobile & cloud"
* Menu
  * Items for all the 'Content:' top-level bullet points below
  * Call to action: 'Ask for Offer'
  * Language selector
  * Facebook link
* Jumbotron
  * Fancy background image
  * "Custom software for small and large businesses and individuals"
* Content: Areas of Expertise
  * Question answered: What do we offer?
  * Lead text should focus on:
    * We design, develop and support software tailored to custom needs
    * Here are our main areas of expertise where we can deliver
  * 2x3 matrix of areas, each with distinguishing icon and short description.
    * Web apps
      * Working with latest standards (HTML5, ES6 JS, CSS3)
      * Using well-known frameworks
      * All apps are designed with a 'mobile-first' UX in mind
    * Mobile apps
      * Targeting Android and iOS
      * Including the development of mobile UX for these two platforms, in accordance with platform-specific guidelines
    * Database development
      * RDMS: Oracle, PostgreSQL, MySQL, etc..
      * Also working with NoSQL solutions
      * Database design from ground-up, modeling of problem domain
      * Performance optimization
    * Cloud and Microservices
      * Implementing server-side components as microservices, as needed
      * Leveraging containerization (Docker, Kubernetes, ...) which allows apps to be deployed easily to cloud as well as to on-premise setups
      * Working with GCP, AWS, Microsoft Azure, etc.; just mention these
    * Reverse Engineering
      * Taking over existing code bases to perform additional development or full/partial rewrite
      * Analizing existing apps to extract and reverse engineer business logic, do post-documentation
    * Support and Consulting
      * Custom support agreements for all our software
      * Consulting to help IT-related decision making, including performing the necessary analysis on customer infrastructure
* Content: How we work?
  * Question answered: How we work?
  * Key message: description of work flow, what customer can expect
  * The workflow:
    1. Customer contacts us via mail or 'Ask for offer' page (call to action, again)
    1. We organize meeting, discuss the details
    1. We create a draft of the concept, if necessary
    1. Agile is at the center: we deliver periodically, depending on agreement with customer
    1. Testing, verification and validation of the software being built is continuous
    1. Constantly reviewing the completeness criteria after each iteration (established together at the beginning); emphasize flexibility
* Content: About
  * Key message: Company introduction, year of foundation, etc..
* Contenct: Contact
  * Company email
  * Company phone number
* Footer
  * Impressum
  * Licences of used materials

## **Ask for Offer** page structure

* Same menu as landing page, linking back to it
* Section: what you need
  * Here one can select the kinds of software needed
  * Sliding (touch-enabled) buttons to express requirements shortly:
    * Mobile application development
    * Web application development
    * Database development
    * Targeting the cloud
    * Development & support of existing software
    * Reverse Engineering of existing software
* Section: contact details form
  * Email
  * Phone
  * If phone is given: when can we contact you (weekday & hour range selector)
* Button to send the request with; feedback of successful sending
